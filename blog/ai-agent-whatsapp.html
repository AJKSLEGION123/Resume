Как я построил AI-менеджера для WhatsApp за 2 недели

20 марта 2026 10 мин чтения AI-агенты Кейс

Клиент — компания в HoReCa B2B. 200+ заявок в месяц через WhatsApp. 3 менеджера копируют данные из чатов в 1С. Среднее время ответа: 15 минут. Задача: автоматизировать.

Проблема

Менеджеры тратили 80% времени на рутину:

  • Копировали заявки из WhatsApp в Excel
  • Проверяли остатки в 1С вручную
  • Формировали КП в Word и отправляли обратно
  • Среднее время ответа: 15 минут
Клиент терял заявки, потому что конкуренты отвечали быстрее.

Архитектура решения

Я собрал систему из 4 компонентов:

  1. WhatsApp Business API — приём входящих сообщений через webhook
  2. Make (Integromat) — маршрутизация и оркестрация
  3. GPT-4o API — понимание запроса, извлечение параметров (товар, количество, сроки)
  4. 1С REST API — проверка остатков, формирование КП, создание счёта

Как работает

Клиент пишет в WhatsApp: Нужно 50 кг курицы и 30 кг говядины

AI-агент:

  1. Распознаёт товары и количество через GPT-4o
  2. Проверяет остатки в 1С через API
  3. Формирует КП с ценами и условиями
  4. Отправляет клиенту в WhatsApp за 10 секунд

Технические решения

Промпт для GPT-4o

Ключевой момент — structured output. GPT возвращает JSON с полями items[], quantity, unit. Это позволяет автоматически маппить на номенклатуру 1С.

Обработка ошибок

Если GPT не может распарсить запрос — бот уточняет: "Уточните, пожалуйста, какой товар и в каком количестве вам нужен?" Если 3 попытки не помогли — переводит на живого менеджера.

Интеграция с 1С

1С выставляет REST API через HTTP-сервис. Make вызывает его для:

  • Проверки остатков по SKU
  • Создания коммерческого предложения
  • Автоматического создания счёта

Результаты за первый месяц

60% заявок обрабатываются полностью без оператора

15 мин → 10 сек — время ответа клиенту

0 пропущенных заявок — бот работает 24/7

Окупаемость за 1 месяц — экономия на 1.5 операторах

Уроки

  1. Начинай с MVP — первая версия работала через 5 дней. Остальные 9 дней ушли на edge cases и тестирование.
  2. Structured output > свободный текст — GPT должен возвращать JSON, а не прозу.
  3. Fallback на человека обязателен — AI не заменяет менеджера, а разгружает его.
  4. Мониторинг с первого дня — логирую каждый запрос и ответ для аналитики.
